PContract MVP — User Flows & Stories

Content-creator escrow platform. Creator drafts a lightweight contract, brand pays upfront into escrow, creator gets brand approval before going live, and payment auto-releases after a 3-day review window (unless disputed). Each job below is framed as Main Job → User Stories → Flow Diagram.

End-to-end journey at a glance

Primary user: Content creator (not legal-savvy, wants fast, frictionless deals, biggest fear = not getting paid). Secondary user: brand contact.

Job 1 — Create & Share a Contract

Actor: Creator · Goal: draft a contract and get a shareable payment link in under 60 seconds.

User Stories

As a creator, I want a single "Create Contract" CTA on the landing page so that I can start without reading onboarding copy. Acceptance: homepage shows value prop + one primary CTA; no signup wall blocks drafting.
As a creator, I want a minimal form (brand name, brand email, deliverables, delivery date, price) so that I can draft the contract in under a minute. Acceptance: five fields only; price validated as currency; delivery date optional.
As a creator, I want legal terms, escrow logic and payment rules auto-applied so that I never have to write them myself. Acceptance: no legal input visible in the form; standard T&Cs linked at the bottom of the generated contract.
As a creator, I want to generate one shareable link after submitting so that I can send it by Copy / Email / WhatsApp. Acceptance: success screen shows the link plus three share buttons; link resolves for a logged-out brand.
As a creator, I want the contract to be locked once sent so that the brand always sees the exact terms I shared. Acceptance: sent contracts are read-only; only option is "Cancel Contract" and create a new one.
As a creator, I want a "moment of confidence" screen so that I feel the deal is now protected. Acceptance: success screen uses language like "Your contract is ready" + visible escrow trust signal.

Job 2 — Brand Reviews & Pays into Escrow

Actor: Brand (external, no signup initially) · Goal: understand the deal in seconds and pay with trust.

User Stories

As a brand, I want to open the contract link without creating an account so that reviewing is frictionless. Acceptance: link works without login; brand email is prefilled from the contract.
As a brand, I want a clean, scannable summary (creator, deliverables, price, timeline) so that I can understand the deal in under 30 seconds. Acceptance: all four data points above the fold; legal T&Cs collapsible.
As a brand, I want a clear trust signal ("Payment held securely in escrow") so that I know my money is not going directly to the creator's wallet. Acceptance: escrow badge visible next to the price; short explainer tooltip available.
As a brand, I want to accept and pay in one flow (card for v2, bank transfer for MVP) so that I don't bounce between screens. Acceptance: "Accept & Pay" button leads directly to payment; on success, confirmation says "Payment secured. Awaiting delivery."
As a brand, I want a clear error if payment fails and a way to retry with another method so that I don't lose my place. Acceptance: failure message "Payment failed. Try another method."; contract stays "Pending Payment" (no activation) until payment succeeds.
As a creator, I want to see "Awaiting Payment" and be able to resend the link / send a reminder if the brand never opens it. Acceptance: creator status is "Awaiting Payment"; actions available: Resend link, Copy link again, Send reminder email.

Job 3 — Pre-Approval & Submit Delivery

Actor: Creator · Goal: get the brand's explicit approval before going live, then submit the public link.

User Stories

As a creator, I want a persistent banner telling me to get content approved before posting so that I don't accidentally forfeit payment protection. Acceptance: banner text: "To guarantee your payment, get the content approved by the brand before posting."; dismissible only temporarily.
As a creator, I want a 3-step approval checklist (Shared with brand → Approval received → Ready to publish) so that I always know the next step. Acceptance: checklist visible on the contract page until delivery is submitted.
As a creator, I want notifications after payment reminding me to share my draft, and before delivery reminding me to get approval. Acceptance: notification 1 fires on "Payment Secured"; notification 2 fires at T–24h of delivery date.
As a creator, I want to submit the live public link on the platform so that the 3-day review window starts cleanly. Acceptance: form accepts a URL; submit CTA = "Submit Delivery".
As a creator, I must confirm via mandatory checkbox that the brand approved the content before publishing so that the escrow logic is protected. Acceptance: submission is blocked until the checkbox "I confirm this content was approved by the brand before going live" is ticked.
As a creator, I want to optionally upload proof (screenshot / message) so that I have evidence if a dispute is raised. Acceptance: optional file upload field; accepted types = image/*, pdf; max 10 MB.
As a creator, I want the status to change to "Under Review (3 days)" on submission so that the countdown is visible. Acceptance: after submit, status flips; a countdown "Auto-release in: Xd Yh" is rendered.

Job 4 — Brand Reviews & Payment Releases

Actors: Brand (active) + Creator (passive) · Goal: during a 3-day window, brand accepts, raises an issue, or does nothing (auto-release).

User Stories

As a brand, I want three clear options in the review window (Accept / Raise Issue / Do nothing) so that I feel fair and controlled. Acceptance: review screen shows the two buttons; "do nothing" is implicit via the countdown.
As a brand, I want to accept in one click and release payment instantly so that I can close the loop when everything is fine. Acceptance: "Accept" triggers release; status flips to "Payment Released"; both sides notified.
As a brand, I want a highly visible countdown ("Auto-release in: 2d 4h") and a warning notification so that I'm never surprised by auto-release. Acceptance: countdown visible top of contract page; notification at T–48h and T–12h; copy: "Payment will be released soon unless you take action."
As a creator, I want a "Payment scheduled for release" notification so that I know the funds are coming. Acceptance: notification at T–48h; status copy: "Payment scheduled for release".
As a creator, I want a celebratory confirmation when payment is released so that the emotional payoff matches the effort. Acceptance: UI shows "Payment released 🎉"; funds move to creator account.
As a brand, if the creator never delivers by the contract deadline, I want a "Raise Issue" prompt so that I can reclaim my money. Acceptance: if delivery deadline passes without submission, the brand sees a "Report no delivery" CTA; confirmed no-delivery = full refund.

Job 5 — Resolve a Dispute

Actors: Brand & Creator · Core rule: the platform does NOT resolve creative disputes. Parties resolve externally and then both must confirm the outcome on-platform.

User Stories

As a brand, I want to open a dispute by selecting a reason (Not as agreed / Late delivery / Other) + an optional comment so that the creator knows why. Acceptance: "Raise Issue" form requires a dropdown reason; optional free-text comment; payment immediately frozen on submit.
As a creator, I want an instant notification when an issue is raised so that I can follow up off-platform fast. Acceptance: in-app + email notification: "⚠️ Brand has raised an issue"; funds show as frozen in status.
As a creator or brand, I want a clear on-screen explanation that we resolve externally and then confirm here so that I don't expect platform arbitration. Acceptance: dispute screen copy: "Please resolve this directly with the other party and confirm the outcome here."
As a creator, I want a "Confirm Agreement" button to release payment so that once we agree off-platform, one click finishes it. Acceptance: payment only releases when BOTH parties have clicked their respective confirm buttons.
As a brand, I want a "No Agreement" button that cancels the contract and refunds me so that I have a clean exit if we can't agree. Acceptance: warning copy: "If no agreement is reached, the contract will be cancelled and the payment returned to the brand."; on confirm = refund.
As both parties, I want a progress indicator showing who has confirmed so that expectations are explicit. Acceptance: UI shows "Creator: ✅ Confirmed / Brand: ⏳ Pending" (or equivalent).
As the system, after 10 days of deadlock with no bilateral agreement, I want to prompt cancellation, and default to refund-to-brand if still no action. Acceptance: 10-day timer; prompt "No agreement reached. Do you want to cancel?"; on timeout → contract cancelled, payment refunded.

Job 6 — Handle Edge Cases

Actors: Both · Goal: money is always safe, status is always clear, next action is always obvious — even when things go wrong.

User Stories

As a brand, when my payment fails, I want a clear retry path so that I don't lose momentum (MVP = bank transfer only). Acceptance: error "Payment failed. Try another method."; contract stays "Pending Payment" until success.
As a creator, when the brand never opens my link, I want to resend / copy the link / send a reminder email so that I can nudge without friction. Acceptance: three actions visible on the contract page while status = "Awaiting Payment".
As a creator, when I realise I entered wrong details (e.g. wrong price), I want to cancel and create a new contract — not edit a sent one — so that trust is preserved. Acceptance: sent contracts are locked; only available write-action is "Cancel Contract"; cancelled contracts cannot be reactivated.
As a brand, when the creator misses the delivery deadline, I want a "Raise Issue" prompt that leads to a full refund on confirmed no-delivery. Acceptance: after deadline + grace period, brand CTA = "Report no delivery"; on creator silence or confirmation, payment fully refunded.
As either party, when I suspect fraud or abuse, I want a "Report issue" entry point so that the platform can flag the contract internally. Acceptance: reporting is one click from the contract page; submission freezes payment for review; no complex investigation UI in MVP.
As a user, at any moment I want to know where the money is, the current status, and the next action so that I'm never lost. Acceptance: every contract page shows: money location (Escrow / With brand / Released / Refunded), status chip, and a primary CTA.
